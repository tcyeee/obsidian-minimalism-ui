This file provides guidance for Claude Code when working in this project.

## Project Overview

**obsidian-minimalism-ui** is an Obsidian plugin that transforms Obsidian into a macOS-native-style application:
- A fixed left info panel displaying current note metadata (title, tags, date, word count, links)
- The left panel is kept open by the plugin at all times — users cannot close or drag it away
- All appearance settings are managed in the settings page (Settings → Community Plugins → Minimalism UI)

## File Structure

```
obsidian-minimalism-ui/
├── main.ts              # Plugin logic (TypeScript source)
├── main.js              # Build output (generated by esbuild, must be committed)
├── styles.css           # Plugin styles (auto-loaded by Obsidian)
├── manifest.json        # Plugin metadata (id/name/version/minAppVersion)
├── versions.json        # Version compatibility map
├── customer-ui.css      # Legacy CSS snippet (kept for reference, not part of the plugin build)
├── esbuild.config.mjs   # Build configuration
├── package.json
└── tsconfig.json
```

## Dev Commands

```bash
npm install        # Install dependencies
npm run dev        # Watch mode — rebuilds main.js on changes to main.ts
npm run build      # Production build
```

## Build Output

Obsidian requires these three files to load a plugin:
- `main.js` — compiled plugin code
- `manifest.json` — plugin metadata
- `styles.css` — styles

**`main.js` must be committed to Git** — users installing from GitHub use the build output directly.

## Key APIs

| API | Purpose |
|---|---|
| `Plugin` | Base class for the plugin |
| `ItemView` | Custom left panel view |
| `PluginSettingTab` | Settings page |
| `workspace.on('layout-change')` | Re-open the panel if it gets closed |
| `metadataCache.getFileCache()` | Read note tags, frontmatter, and links |
| `(metadataCache as any).getBacklinksForFile()` | Get backlinks (internal API, no type definition) |

## Notes

- `getBacklinksForFile` is an internal Obsidian API with no type definition — access it via `as any`
- The plugin intercepts `Cmd+\` / `Ctrl+\` via a `keydown` listener (capture phase) to prevent the left sidebar from being closed
- Obsidian uses a three-column layout (left + main + right); the right column is hidden via CSS to achieve a two-column appearance
- CSS variable overrides in `styles.css` rely on Obsidian's default theme variable names — third-party themes may require adaptation
